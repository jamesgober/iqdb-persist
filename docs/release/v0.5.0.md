# iqdb-persist v0.5.0 — API freeze + adversarial hardening

**The surface is committed.** v0.5.0 freezes the public API and the on-disk
format, and hardens the parse and recovery paths against hostile and
partial-write input. With the feature set already frozen at v0.4, this is
the last design-bearing release before the 0.6–0.9 alpha/beta/rc series and
1.0 — what you build against now keeps compiling and loading.

## What is iqdb-persist?

The embedded persistence layer of the iQDB vector database: durable snapshot
save/load for any `iqdb_index::Index`, behind a versioned header, a CRC32
integrity check, and an atomic write; an optional write-ahead log that makes
mutations between snapshots crash-recoverable; and optional Zstd/LZ4
snapshot compression.

## Frozen surface

The committed SemVer 1.x API (full list in `dev/ROADMAP.md`):

- **`Persistable`** — `INDEX_TYPE`, `save_to`, `load_from`.
- **`PersistedIndex<I>`** — `open_with`, `load`, `index`, `index_mut`,
  `config`, `insert`, `delete`, `save`, `checkpoint`.
- **`PersistConfig`** (`path`, `wal_enabled`, `fsync_policy`,
  `compression`) + `new` / `Default`; **`FsyncPolicy`**; **`Compression`**.
- **`PersistError`** (`#[non_exhaustive]`) + **`Result`**.
- **`FileHeader`**, **`MAGIC`**, **`CURRENT_VERSION`**, **`format`**
  (`read_header` / `write_header`), **`checksum`** (`compute` / `verify`),
  **`VERSION`**.

The **on-disk format is frozen** too: snapshot format version 2 (header +
compression preamble + payload CRC32), the WAL framing (`IQDBWAL\0`,
per-record CRC32), and the metric-tag mapping. Format v1 stays readable.
Any future change goes through a version bump — never a silent
reinterpretation.

## Adversarial hardening

`tests/adversarial.rs` exhaustively attacks the loader:

- **Every single-byte flip** of a valid snapshot is rejected — never a
  panic, never a silently-different load. Every byte is either header
  metadata (validated or cross-checked against the reconstructed index) or
  inside the CRC-covered payload region.
- **Every truncation length** of a snapshot is rejected.
- **Every single-byte flip and every truncation of a WAL** is handled: a
  corrupt frame stops replay at the last intact record, a torn tail is
  dropped, a corrupt WAL header is a clean `Err` — and the recovered index
  never holds more records than were written.
- **Pseudo-random garbage** (deterministic SplitMix64) as a snapshot or a
  WAL never panics or over-allocates.

These join the existing atomicity test (a failed rename leaves the prior
snapshot byte-for-byte intact) and the WAL crash-recovery tests to cover the
directives' contract: untrusted input is validated, allocations are bounded,
and library code does not panic on hostile input.

## Deferred: `storage-io` integration

The roadmap's v0.5 also named `storage-io` integration. `storage-io` is the
*renamed* `fsys-rs`, and **that rename has not happened** — the crate is
still `fsys` (1.1.0). Rather than couple to a name that is about to change,
integration is **deferred** (recorded in `dev/ROADMAP.md`): the internal
`Storage` trait is already the swap seam, so adopting the substrate later is
an internal change, not an API break. The substrate itself remains out of
scope for 1.0.

## Verification

- `fmt`, `clippy -D warnings` (default and `--all-features`),
  `test` (default and `--all-features`), and `cargo doc -D warnings` are
  clean; builds on stable and the 1.87 MSRV; `cargo deny` + `cargo audit`
  clean. Verified on **Windows and WSL2/Linux**.
- **Zero `unsafe`** (`#![forbid(unsafe_code)]`); no `unwrap` / `expect` /
  `todo!` / `dbg!` in shipping code.

## Breaking changes

**None.** This release only adds tests and freezes the existing surface. The
API and on-disk format are unchanged from v0.4 — now committed.

## What's next

- **v0.6.x–0.7.x (alpha)** — integrate against real iQDB consumers;
  MINOR-compatible additions only.
- **v0.8.x (beta)** — bug fixes, broader testing, final benchmarks.
- **v0.9.x (rc)** — critical fixes and doc polish, then **1.0**.

## Installation

```toml
[dependencies]
iqdb-persist = "0.5"               # or: features = ["zstd", "lz4"]
iqdb-index   = "1.0"
iqdb-types   = "1.0"
```
