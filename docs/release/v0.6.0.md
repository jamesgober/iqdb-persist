# iqdb-persist v0.6.0 — Alpha: core-invariant property tests

**Proving the contract.** With the feature set, public API, and on-disk
format all frozen (v0.4–v0.5), v0.6.0 opens the alpha series by property-
testing the two invariants the whole crate rests on, and shipping an
end-to-end recovery example. No surface changes — this is confidence.

## What is iqdb-persist?

The embedded persistence layer of the iQDB vector database: durable snapshot
save/load for any `iqdb_index::Index`, behind a versioned header, a CRC32
integrity check, and an atomic write; an optional write-ahead log that makes
mutations between snapshots crash-recoverable; and optional Zstd/LZ4
snapshot compression.

## The two invariants, now property-tested

`tests/invariants.rs` runs both `dev/DIRECTIVES.md` §8 guarantees against a
full-fidelity mock index that serialises ids, vectors, **and** metadata —
so the comparison exercises the entire pipeline, not a toy subset:

1. **Atomic snapshot round-trip.** For an arbitrary set of rows, `save`
   followed by `load` reconstructs identical state — checked across every
   compression scheme compiled into the build (`None`, and `Zstd` / `Lz4`
   when their features are on).
2. **WAL replay is the recovery contract.** For an arbitrary sequence of
   `insert` / `delete` / `checkpoint` operations, dropping the index
   without a trailing checkpoint (a simulated crash) and then calling
   `load` recovers **exactly** the state an equivalent in-memory model
   holds. This is the strongest statement of the durability promise:
   whatever you acknowledged, you get back.

These join the existing header round-trip proptest, the WAL record-encoding
proptest, the adversarial byte-flip / truncation / garbage suite, and the
atomicity test into a coverage set that spans the format, the codecs, the
log, and recovery.

## End-to-end example

`examples/wal_recovery.rs` walks the durable-mutation lifecycle a real
consumer follows — enable the WAL, `insert` / `delete` through the wrapper,
`checkpoint` to fold the log into a snapshot, simulate a crash, and recover
the last acknowledged state:

```sh
cargo run --example wal_recovery
# session 1 (in memory): 2 vectors
# session 2 (recovered): 2 vectors, ids = [2, 3]
```

## A note on "real consumers"

The roadmap's alpha goal is integration against real index consumers. A
concrete `Persistable` impl for `iqdb-flat` / `iqdb-hnsw` / `iqdb-ivf`
cannot live in this crate: those 1.0 crates expose no way to enumerate their
stored entries and do not depend on `iqdb-persist`. That wiring belongs in
the umbrella `iqdb` crate — which already owns the `index_type` →
concrete-type registry a create-or-load call needs. Within this repo the
trait contract is validated by the full-fidelity mock and the runnable
examples; the impl pattern they demonstrate is exactly what the umbrella
crate will follow.

- `fmt`, `clippy -D warnings` (default and `--all-features`),
  `test` (default and `--all-features`), and `cargo doc -D warnings` are
  clean; builds on stable and the 1.87 MSRV; `cargo deny` + `cargo audit`
  clean. Verified on **Windows and WSL2/Linux**.
- **Zero `unsafe`** (`#![forbid(unsafe_code)]`); no `unwrap` / `expect` /
  `todo!` / `dbg!` in shipping code.

## Breaking changes

**None.** Tests, an example, and docs only. API and on-disk format unchanged
since v0.4, frozen since v0.5.
