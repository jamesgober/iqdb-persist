# iqdb-persist v1.0.0 — Stable

**Persistence is committed.** v1.0.0 is the first stable release of the iQDB
disk-persistence layer. The feature set is complete, the public API and the
on-disk format are frozen under the SemVer 1.x guarantee — no breaking
changes until 2.0 — and what you build and write today keeps compiling and
loading.

## What is iqdb-persist?

The embedded persistence layer of the iQDB vector database — what moves iQDB
from demo-only to actually usable. It adds durable snapshot **save** and
**load** to any index that implements `iqdb_index::Index`, behind a versioned
header, a CRC32 integrity check, and an atomic write; an optional write-ahead
log makes mutations between snapshots crash-recoverable; and snapshots can be
transparently compressed. It is generic over the index — the framing lives
here, the payload lives in the index's `Persistable` impl — so it persists
`iqdb-flat`, `iqdb-hnsw`, `iqdb-ivf`, and any future index uniformly.

## The 1.0 surface

```rust
use iqdb_persist::{PersistConfig, PersistedIndex};

// `MyIndex: iqdb_index::Index + iqdb_persist::Persistable`
let mut cfg = PersistConfig::new("/var/lib/app/index.iqdb");

// Snapshot-only: wrap, save atomically, load back (verifies magic,
// version, type, CRC32).
PersistedIndex::open_with(my_index, cfg.clone())?.save()?;
let restored: PersistedIndex<MyIndex> = PersistedIndex::load(cfg.clone())?;

// Or durable, crash-recoverable mutation:
cfg.wal_enabled = true;
let mut db = PersistedIndex::open_with(other_index, cfg.clone())?;
db.insert(id, vector, metadata)?;   // logged + fsynced, then applied
db.delete(&old_id)?;
db.checkpoint()?;                    // fold the WAL into a fresh snapshot
// after a crash: PersistedIndex::load(cfg) replays the WAL onto the snapshot
```

The committed surface:

- **`Persistable`** — `INDEX_TYPE`, `save_to`, `load_from`.
- **`PersistedIndex<I>`** — `open_with`, `load`, `index`, `index_mut`,
  `config`, `insert`, `delete`, `save`, `checkpoint`.
- **`PersistConfig`** (`path`, `wal_enabled`, `fsync_policy`,
  `compression`) + `new` / `Default`; **`FsyncPolicy`**; **`Compression`**.
- **`PersistError`** (`#[non_exhaustive]`) + **`Result`**.
- **`FileHeader`**, **`MAGIC`**, **`CURRENT_VERSION`**, **`format`**
  (`read_header` / `write_header`), **`checksum`** (`compute` / `verify`),
  **`VERSION`**.

## Highlights

- **Atomic by construction.** Every snapshot write is temp-file + `fsync` +
  atomic rename + (POSIX) directory `fsync`; a failure at any step leaves the
  prior snapshot byte-for-byte intact.
- **Crash-recoverable.** With the WAL on, an acknowledged mutation is durable
  before it touches memory and is replayed on load; a torn tail from a crash
  mid-append is detected and discarded.
- **Portable format.** Fixed-width little-endian throughout — a file written
  on a 64-bit host reads on a 32-bit one. Format version 2; version-1 files
  still load.
- **Optional compression.** Zstd (best ratio) or LZ4 (fastest) on the
  snapshot payload, behind cargo features; the CRC32 covers the compressed
  bytes, so corruption is caught before decompression.
- **Hardened.** Exhaustive single-byte-flip / truncation / garbage testing of
  snapshot and WAL parsing; the loader never panics, never over-allocates,
  never returns a silently-wrong result.

## Why this is 1.0

Every Definition-of-Done criterion (`dev/DIRECTIVES.md` §7) is met:

- **Clean everywhere** — `fmt`, `clippy -D warnings` (default and all
  features), `test` (default and all features), and `cargo doc -D warnings`
  pass on the CI matrix (Linux, macOS, Windows), on stable and the 1.87 MSRV.
  Verified locally on Windows and WSL2/Linux.
- **Secure** — `cargo audit` + `cargo deny` clean; **zero `unsafe`**
  (`#![forbid(unsafe_code)]`); no `unwrap` / `expect` / `todo!` / `dbg!` in
  shipping code; every allocation bounded; no panic on hostile or
  partial-write input (the adversarial suite proves it).
- **Correct** — both §8 invariants are property-tested: snapshot save→load
  preserves state across every compression scheme, and a random
  insert/delete/checkpoint sequence followed by a crash recovers exactly the
  in-memory model's state.
- **Documented** — every public item carries a runnable rustdoc example; the
  README leads with the Tier-1 surface; `docs/API.md` covers the whole API;
  `save_and_load` and `wal_recovery` examples ship in `examples/`.
- **Benchmarked** — `criterion` benches cover the hot paths (WAL append and
  replay; Zstd/LZ4 compress and decompress).

`loom` is **not applicable**: `PersistedIndex` is single-writer (`&mut self`
for mutation, `&self` for reads) with no shared-state or lock-free path, so
there is nothing for a concurrency model checker to explore.

## Breaking changes

**None.** This is the first stable tag; the surface is now committed under
the 1.x guarantee. The API and on-disk format are unchanged from v0.6.

## Out of scope for 1.0 (additive, post-1.0)

- **Concrete `Persistable` impls** for `iqdb-flat` / `iqdb-hnsw` /
  `iqdb-ivf` — these live in the umbrella `iqdb` crate (which owns the
  `index_type` → concrete-type registry), since the published index crates
  expose no entry enumeration and do not depend on `iqdb-persist`.
- **`storage-io` integration** — the renamed `fsys-rs` substrate. The
  internal `Storage` seam is ready; adopting it later is an internal swap,
  not an API break.
