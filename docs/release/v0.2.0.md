# iqdb-persist v0.2.0 — Atomic snapshots

**Persistence is real now.** v0.2.0 delivers the hard part of the roadmap —
the versioned on-disk format, atomic snapshot save/load, and CRC32
integrity — onto the v0.1.0 scaffold. An iQDB index can now be written to
disk and read back, and an interrupted write can never corrupt an existing
good file.

## What is iqdb-persist?

The embedded persistence layer of the iQDB vector database. It adds durable
snapshot **save** and **load** to any index that implements
`iqdb_index::Index`, wrapped behind a small framing layer (versioned file
header + CRC32) and an atomic write (temp file + `fsync` + rename +
directory `fsync`). It never names a concrete index — it is generic over
`I: Index + Persistable` — so it persists `iqdb-flat`, `iqdb-hnsw`,
`iqdb-ivf`, and any future index uniformly.

## The surface

```rust
use iqdb_persist::{PersistConfig, PersistedIndex};

// `MyIndex: iqdb_index::Index + iqdb_persist::Persistable`
let cfg = PersistConfig::new("/var/lib/app/index.iqdb");

// Wrap an in-memory index and save it atomically.
PersistedIndex::open_with(my_index, cfg.clone())?.save()?;

// Later — reconstruct from disk (verifies magic, version, type, CRC32).
let restored: PersistedIndex<MyIndex> = PersistedIndex::load(cfg)?;
```

An index opts in by implementing the two-method `Persistable` trait
(`save_to` / `load_from` + a stable `INDEX_TYPE` tag). A complete, runnable
version ships in [`examples/save_and_load.rs`](../../examples/save_and_load.rs).

The v0.2 public surface:

- **`Persistable`** — the trait an index implements; payload-only, with the
  framing added around it.
- **`PersistedIndex<I>`** — `open_with`, `load`, `save`, and the `index` /
  `index_mut` / `config` accessors.
- **`PersistConfig` / `FsyncPolicy` / `Compression`** — configuration; the
  `wal_enabled` and compression knobs reject cleanly until they ship.
- **`FileHeader` / `MAGIC` / `CURRENT_VERSION`** plus
  `format::{read_header, write_header}` and `checksum::{compute, verify}` —
  for tooling that inspects a snapshot without loading it.
- **`PersistError`** — `#[non_exhaustive]`, `error_forge::ForgeError`-
  integrated, one variant per failure mode.

## Highlights

- **Atomic by construction.** Save writes a temp file, `fsync`s it, renames
  it over the target, and (on POSIX) `fsync`s the directory. A failure at
  any step leaves the original byte-for-byte intact — proven by a test that
  injects a rename failure and confirms the prior snapshot survives.
- **Portable format.** Every size is fixed-width little-endian `u64`; a file
  written on a 64-bit host reads on a 32-bit one. The `index_type` length is
  capped at 4 KiB so a hostile header cannot trigger a giant allocation.
- **Corruption is caught, not propagated.** A single-bit flip in the payload
  surfaces as `ChecksumMismatch`; a header that disagrees with the
  reconstructed index's `dim` / `metric` / `len` surfaces as
  `InvalidPayload`; the wrong index type surfaces as `InvalidIndexType` —
  never a panic, never a silently-wrong load.
- **Substrate seam.** All file I/O goes through an internal `Storage` trait
  so the future `storage-io` crate can drop in unchanged.

## Verification

- `fmt`, `clippy -D warnings` (default and `--all-features`),
  `test --all-features`, and `cargo doc -D warnings` are clean; builds on
  stable and the 1.87 MSRV.
- **Zero `unsafe`** (`#![forbid(unsafe_code)]`); no `unwrap` / `expect` /
  `todo!` / `dbg!` in shipping code.
- Property-tested header round-trip (128 cases) plus targeted error-shape
  tests (bad magic, wrong version, truncation, invalid metric); unit tests
  for atomic-save survival, CRC mismatch, index-type mismatch, and config
  validation.

## Breaking changes

**None** relative to v0.1.0 (the scaffold had no surface). The on-disk
format is version `1`; it is **not yet frozen** — the format freeze lands at
the v0.5.0 API freeze, per the roadmap.

## What's next

- **v0.3.0 — WAL append + replay + crash recovery.** Every mutation logged
  before it is applied in memory, replayed on startup.
- **v0.4.0 — Zstd / LZ4 compression + feature freeze.**
- **v0.5.0 — `storage-io` integration, adversarial/partial-write tests, and
  the public-API + on-disk-format freeze.**
